# 產品需求文件 (PRD)：個人記帳簿系統

## 1. 專案概述

### 背景與動機
現代人生活節奏快、消費管道多元，許多人因為缺乏有效的記帳習慣，到了月底往往不知道錢花到哪裡去，容易成為「月光族」。為了解決這個痛點，本專案旨在建立一個操作簡單且功能完整的個人記帳工具，幫助使用者釐清每日金流、培養健康的理財習慣。

### 目標用戶
- 希望掌握每月收支狀況與財務的學生和上班族。
- 想要開始存錢或進行理財規劃，需要追蹤資產的個人。
- 覺得市面上現有記帳 App 太過複雜，追求簡單直覺介面的使用者。

### 核心價值主張
提供化繁為簡的記帳體驗，結合直覺的視覺化圖表、預算設定功能與多帳戶資產追蹤，幫助使用者無痛管理個人財富。

---

## 2. 功能需求

### 1) 紀錄每日開銷
- **使用者故事**：作為 **一般用戶**，我希望 **能快速且方便地輸入每筆消費、收入或轉帳明細**，以便 **完整記錄我的日常資金流向**。

### 2) 分析消費習慣
- **使用者故事**：作為 **一般用戶**，我希望 **能透過圓餅圖或長條圖看見各類別的花費佔比**，以便 **了解我的消費結構並找出可以節省的項目**。

### 3) 預算控制
- **使用者故事**：作為 **一般用戶**，我希望 **能設定每月總預算及不同類別的預算額度，並在快滿額時獲得提醒**，以便 **有效控制花費避免超支**。

### 4) 統計分析
- **使用者故事**：作為 **一般用戶**，我希望 **能按週、月或年查看歷史收支趨勢折線圖與總和統計**，以便 **檢視長期以來的財務健康度與變化**。

### 5) 資產餘額追蹤
- **使用者故事**：作為 **一般用戶**，我希望 **能建立不同的資金帳戶（如現金、銀行帳戶、信用卡等）並查看總資產餘額**，以便 **清楚掌握我手邊可動用的資金與負債狀況**。

---

## 3. 非功能需求

### 技術限制
- **後端框架**：使用 Python 的 Flask 框架。
- **前端模板**：使用 Jinja2 搭配 HTML/CSS/JavaScript。
- **資料庫**：使用 SQLite，適合輕量級應用與快速開發。

### 效能與安全考量
- **安全性**：表單輸入需防止 SQL Injection 及 XSS 攻擊；若包含會員登入機制，密碼必須經過加密（如 Bcrypt）處理儲存。
- **效能**：資料庫查詢需進行基本優化，確保在數千筆記帳紀錄下，頁面載入與統計報表生成時間仍能保持順暢。
- **易用性**：介面需具備良好的防呆機制（如：金額不可為負數、未填寫必填欄位時給予明確提示），並支援 RWD 以適應桌機與手機網頁瀏覽。

## 4. MVP 範圍

### Must Have (必要功能)
- 紀錄每日開銷（新增、讀取、修改、刪除記帳明細）。
- 基礎統計分析（顯示當月收入總計、支出總計與結餘）。
- 資產餘額追蹤（單一或基礎帳戶餘額連動計算）。

### Should Have (次要功能)
- 分析消費習慣（以簡易圖表呈現支出類別佔比）。
- 建立多個資產帳戶（現金、銀行等），支援帳戶間內部轉帳。

### Nice to Have (加分功能)
- 預算控制（自訂預算與超支進度條提醒）。
- 定期收支自動紀錄。
- 資料匯出（如 Excel 或 CSV）與匯入功能。
